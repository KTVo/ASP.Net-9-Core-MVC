How to Display Variables Between MVC:
    Not to mention, how a C# project forms a connection between models, views, and controller is that
    the action method (which is methods within the controllers handling incoming HTTP-requests) is the same
    namee as the view file which in this case is "Overview.cshtml".

    Within the Overview.cshtml, it knows which model to use by importing it as such: @model MyApp.Models.Item

    Where within the Controller, you'd declare a new instance of the model class like so:

        public IActionResult Overview()
        {
            Item item = new Item()
            {
                Name = "Keyboard",
                Color = "Yellow"
            };
            return View(item);
        }

IActionResult:
    Finally, I action results is a promise on what the return of a Controller action method should be. The best 
    feature about it is that it has all knowledge of all possible returns for an action method. This includes
    Content, View, etc. and you'll be able to return all such things.

Entity Framework:
    Entity Framework is an ORM (Object-Relational Mapping). So, essentially it connects the C# model class to 
    the database table.

    There is a Code First (Model First) and Database First approach.

    dotnet add package Microsoft.EntityFrameworkCore
    dotnet add package Microsoft.EntityFrameworkCore.Tools
    dotnet add package Microsoft.EntityFrameworkCore.SqlServer

    To migrate your C# models to database (Code First Approach):

            dotnet ef migrations add InitialCreate --context MyAppContext

            dotnet ef database update 

    To migrate your database tables into your C# project as models (Database First Approach):
        PM> <NAME_FOR_DB_CONTEXT> "Data Source=<SQL_SERVER_NAME"; Initial
        Catalog=MenuBlazor_Data; Integrated Security=True;Pooling=False; Encrypt=False;Trust Server Certificate=True" 
        Microsoft.EntityFrameworkCore.SqlServer -ContextDir Data -OutputDir Models - DataAnnotation
